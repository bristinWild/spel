# spel-client-gen

Generate typed Rust client code and C FFI wrappers from SPEL program IDL JSON.

## Overview

`spel-client-gen` reads the IDL JSON that SPEL programs produce (via `#[lez_program]` macro) and generates:

1. **Typed Rust client** ‚Äî a struct with async methods per instruction, correct account ordering, PDA computation helpers, and proper type conversions
2. **C FFI wrappers** ‚Äî `extern "C"` functions accepting/returning JSON strings, matching the pattern used by `lez-multisig-ffi`
3. **C header file** ‚Äî for inclusion in C/C++ projects (e.g., Qt plugins)

## Usage

### As a CLI tool

```bash
cargo run -p spel-client-gen -- --idl path/to/idl.json --out-dir generated/
```

This produces three files:
- `<program>_client.rs` ‚Äî typed Rust client module
- `<program>_ffi.rs` ‚Äî C FFI wrapper
- `<program>.h` ‚Äî C header

### As a library

```rust
use spel_client_gen::generate_from_idl_json;

let idl_json = std::fs::read_to_string("my_program_idl.json")?;
let output = generate_from_idl_json(&idl_json)?;

std::fs::write("src/generated_client.rs", &output.client_code)?;
std::fs::write("src/generated_ffi.rs", &output.ffi_code)?;
std::fs::write("include/my_program.h", &output.header)?;
```

## IDL Input Format

The input is the standard SPEL IDL JSON format generated by the `#[lez_program]` macro:

```json
{
  "version": "0.1.0",
  "name": "my_program",
  "instructions": [
    {
      "name": "create",
      "accounts": [
        {
          "name": "state",
          "writable": true,
          "signer": false,
          "init": true,
          "pda": {
            "seeds": [
              {"kind": "const", "value": "my_state"},
              {"kind": "arg", "path": "key"}
            ]
          }
        },
        {"name": "creator", "writable": false, "signer": true, "init": false}
      ],
      "args": [
        {"name": "key", "type": "[u8; 32]"},
        {"name": "threshold", "type": "u64"}
      ]
    }
  ],
  "accounts": [],
  "types": [],
  "errors": []
}
```

## Generated Output

### Client Code

- `<Program>Instruction` enum matching the IDL
- `<Instruction>Accounts` structs with correctly-typed fields
- `<Program>Client` struct with:
  - One async method per instruction
  - Correct account ordering (matching IDL)
  - Automatic signer detection and key lookup
  - PDA computation helpers (e.g., `compute_state_pda(...)`)

### FFI Code

- One `extern "C"` function per instruction: `<program>_<instruction>(args_json) -> *mut c_char`
- JSON-in / JSON-out pattern matching existing FFI crates
- Proper type parsing:
  - `AccountId`: base58 (native) or hex fallback
  - `ProgramId`: hex ‚Üí `[u32; 8]` with **little-endian** byte order
- PDA computation inline where the IDL specifies seeds
- Memory management: `<program>_free_string()` to free returned strings

## Qt/QML Basecamp Module (`--target logos-module`)

Generates a complete Qt6/QML plugin that can be loaded by [Basecamp](https://github.com/logos-co/basecamp) or run standalone, with a full UI for every instruction and account fetch in the IDL.

### Usage

```bash
# Generate the scaffold (after running make idl)
spel-client-gen \
  --idl my_program-idl.json \
  --out-dir ui/my_program \
  --target logos-module

# Optional: override the class/file prefix (defaults to IDL name)
spel-client-gen \
  --idl my_program-idl.json \
  --out-dir ui/my_program \
  --target logos-module \
  --module-name my_program

# Re-generate after customising qml/Main.qml ‚Äî skips the QML file, rewrites everything else
spel-client-gen \
  --idl my_program-idl.json \
  --out-dir ui/my_program \
  --target logos-module \
  --skip-ui

# Build the standalone preview app
cd ui/my_program
cmake -B build && cmake --build build

# Run it (set env vars or use the Settings page in the UI)
MY_PROGRAM_PROGRAM_ID=<64-char-hex> \
NSSA_WALLET_HOME_DIR=~/.wallet \
NSSA_SEQUENCER_URL=http://127.0.0.1:3040 \
./build/MyProgramApp
```

Or with `make` (if using a project scaffolded by `spel init`):

```bash
make idl         # generate IDL from program source
make ffi         # build FFI .so (ffi-gen ‚Üí cargo build)
make ui-gen      # generate scaffold (wires CMakeLists.txt to FFI .so automatically)
make ui-regen    # re-generate C++ backend only; preserves hand-written qml/Main.qml
make ui-build    # cmake build
make ui-run      # run the standalone preview app
make lgx         # build portable LGX archive for Basecamp distribution
```

### What the UI provides

**Sidebar navigation** groups pages into four sections:

| Section | Pages |
|---|---|
| ACCOUNTS | One page per `#[account_type]` with PDA seeds ‚Äî shows live state, auto-refreshes |
| INSTRUCTIONS | One page per `#[instruction]` ‚Äî typed inputs, enum dropdowns, optional field toggles |
| WALLET | Connection ping, account listing/creation, account inspector |
| SETTINGS | Wallet path, sequencer URL, program ID ‚Äî persisted via `QSettings` |

### IDL annotations that unlock features

| Annotation | Effect on generated UI |
|---|---|
| `#[account_type]` on a struct | Adds a fetch page in the ACCOUNTS section |
| `pda = literal("seed")` | Fetch page has no seed inputs (auto-derived) |
| `pda = arg("key")` | Fetch page shows a seed input field for that arg |
| `Option<T>` arg type | Renders a checkbox-gated field (unchecked = omit from call) |
| Enum type in `idl.types` | Renders a ComboBox instead of a text field |

### Configuration

The backend reads configuration from three sources in priority order:

1. **`QSettings`** (persisted from the Settings page)
2. **Environment variables**: `NSSA_WALLET_HOME_DIR`, `NSSA_SEQUENCER_URL`, `{PROGRAM}_PROGRAM_ID`
3. **Compiled-in FFI constant** from `{prog}_program_id()`

### Wallet page

The wallet page provides:

- **Connection check** ‚Äî pings the sequencer via `get_accounts_nonces([])`
- **Account listing** ‚Äî calls `wallet account list`, shows label + path + ID, click-to-inspect
- **Account creation** ‚Äî calls `wallet account new public [--label <label>]`, refreshes list
- **Account inspector** ‚Äî fetches raw account data, shows size, hex preview, signing-key presence

> **Note:** account listing and creation delegate to the `wallet` CLI binary via subprocess.
> This is a temporary measure until the wallet library exposes these operations natively.
> Remove the `{prefix}_list_accounts` and `{prefix}_create_account` FFI functions and the
> corresponding backend methods once a proper API exists.

### As a library

```rust
use spel_client_gen::{generate_logos_module_from_idl_json, pascal_case};

let idl_json = std::fs::read_to_string("my_program-idl.json")?;
let output = generate_logos_module_from_idl_json(&idl_json, Some("my_program"), None)?;

let class = pascal_case("my_program");
std::fs::write(format!("src/{class}Backend.h"),   &output.backend_h)?;
std::fs::write(format!("src/{class}Backend.cpp"), &output.backend_cpp)?;
std::fs::write(format!("src/{class}Plugin.h"),    &output.plugin_h)?;
std::fs::write(format!("src/{class}Plugin.cpp"),  &output.plugin_cpp)?;
std::fs::write("src/main.cpp",   &output.main_cpp)?;
std::fs::write("qml/Main.qml",   &output.main_qml)?;
std::fs::write("module.yaml",    &output.module_yaml)?;
std::fs::write("manifest.json",  &output.manifest_json)?;
std::fs::write("CMakeLists.txt", &output.cmake_lists)?;
```

## Key Design Decisions

- **Little-endian ProgramId**: `[u32; 8]` words are parsed from hex using `u32::from_le_bytes()`, matching RISC Zero Digest byte order
- **Base58-first AccountId**: FFI functions try base58 parsing first (native format), falling back to hex
- **Account order preserved**: The generated account list exactly matches the IDL order ‚Äî this fixes bugs in hand-written FFI where accounts were in wrong order
- **Rest accounts**: Accounts marked with `"rest": true` become `Vec<AccountId>` and are appended after fixed accounts
